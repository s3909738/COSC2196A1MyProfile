Project Idea:
Overview:

I Propose a project to develop a mobile application, based on the Android platform, that integrates with opensource speech recognition software that would function as a voice driven assistant to make audio journals and generate text transcripts. The application would also be used as an interface to access, share and manage journals. The application would feature the ability to use inline voice commands to manipulate and manage the journal data from within a recording session.
The project would include exploring ways to implement a simple inline voice command language that is recognised by the assistant. With main inline commands aiming to give users the ability to format and insert meta data into their journals on the fly. Making it more convenient for users to access information within audio recordings and transcripts later on. The application would also be used as a brainstorming tool and include the ability to insert links to google search results into journal transcripts.
The project would investigate enhancing the assistant with AI that can implement grammar and punctuation restoration within generated transcripts. And explore cloud-based services the application could link with to upload and share journals.
The project could also have Potential to Investigate Implementing a web-based front end and setting up a server environment to store and manage user accounts and data.

Motivation:

If you haven’t already guessed, I have taken much inspiration from Star Trek, observing how the captains communicate with their ships computer to make logs.
With inspirational ideas easily escaping us without having the chance to write them down, many writers are migrating to using Audio journals. Audio Journals are faster than writing and can help different creative types to brainstorm more freely (MasterClass 2020). This assistant would make recording audio journals and ideas more accessible to researchers, developers and students. At its core, the assistant simply records audio and provides a transcription. But this project would also implement using verbal commands to drive the application and enhance, manipulate, transcriptions and audio recordings. With the user experience being as simple or as advanced to suit their needs.

Description:

The main feature of this project would be using voice commands to control the assistant and initiate inline functions to manipulate transcripts and recorded audio as you are recording. It would also include the ability to share, manage and play back journals, and read transcripts. Main control commands would include:

  • “start”, “stop”, “pause” and “resume” recording functions.
  • Being able to stop and delete the current recording.
  • Play back journals with the ability to start from certain points within the recording.
  • Pull up a list of timestamps within a journal.
  • Start a “supplemental log” that amends to a previous journal
  • Directly share journals to contacts or groups.
Transcript and audio manipulation functions would include:
  • Adding a journal tag that can be used to group it with related journals.
  • Adding a heading with a time stamp before a paragraph or new topic that can be directly pulled up later without having to read through the entire transcript. This could also be used to pull up the audio from that point too.
  • Tagging contacts and groups to your journals.
  • Insert google search links for references and further research.
  • Strike the previous sentence or topic paragraph, in order to start over without mucking up an entire journal.
Inline voice command functions could work similar to a crude form of HTML markup. Using an activating key word before your instruction, to prompt the software of an incoming function. An example of this structure could look like:

“[key word to prompt incoming function command] – [command]-information-[/end command]”

The application would then apply the function and delete out the spoken command words from the audio recording and transcript. This feature would require much brainstorming and development. A drawback to this feature would be relying on user investment to learn the voice activated commands, to make the most of the application. But with just a basic understanding of the applications’ voice driven interface, it could still be used to make general audio journals.

A use case scenario to demonstrate the application in action could be described as follows:
Imagine a user talking to the journal assistant.

“[app prompt] start new journal” – start a new journal entry
“[app prompt] [Title] my first journal [end title]” – add a title
“[app prompt] [heading] my first experience [end heading]” – add a heading that can be jumped back to in later reading
“[app prompt] [contact] Zachary Williams [end contact]” – tag someone from your contact list
“[app prompt] [group] OUA IIT [end group]” – tag a group
“This is my first journal. I feel like a Star Fleet Captain…” – insert body content
“I am going to need to learn Java to make this app…”
“[app prompt] [research] java tutorials [end research]” – provides a google search link into the transcript
“[app prompt] [heading] further research [end heading]” – start a new paragraph and add a new heading that can be jumped back to later
“I need to research opensource speech recognition software and...”
“[app prompt] [research] opensource speech recognition software [end research]”
“[app prompt] [stop recording]” – terminate recording and save audio journal and transcript

The [app prompt] word would need to be a unique word that isn’t going to come up in general conversation, as not to confuse the application.

Other features of the application project would include: accessing and managing audio journals and transcripts, sharing with contacts and providing an in app notification of journals that you have been tagged in and can link to.

Outcome:

Completion of this project would aim to bring easier access to information within audio journals. Providing transcripts, headings and timestamps that can be used to quickly navigate through the information, rather than having to listen through long audio sessions until you have reached the information required. The application would also aim to enhance the availability of audio journals. providing a voice driven interface to request auditory information and a graphical interface for accessing transcripts. The application would also help enable the ability to share with others through inserting contact and group tags. I hope that this application would contribute to sharing inspiration and ideas throughout the academic world. If successful, I would like to see this application integrated with mainstream assistant technologies such as, Google assistant and Alexa, making its way into the home and being used by people and families to share their ideas and spoken stories with one another.
